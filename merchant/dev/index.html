开发者前端契约：API 凭据区（Merchant Code 明文 + Gateway Credential 默认脱敏 / 按需授权 reveal 完整值 / 复制）+ 集成参数区（Gateway URL + Verification Type=RSA2 只读）+ 签名密钥区（商户 RSA2 公钥登记 / 替换 / 查看 / 复制 + 平台公钥查看 / 复制）。 merchant scope 从会话取、不信请求体；全部 op merchant-current-scoped，均挂 MerchantContextGuard（stale-tab 守卫，无豁免）。 全部端点前置 Merchant.status = ACTIVE 门禁（非 ACTIVE → 403；开发者页 = Settings 下仅已激活商户可达，API 凭据在激活后签发）。 字段类型以本 spec 为准。

credentials

API 凭据：Merchant Code + Gateway Credential（默认脱敏 / 授权 reveal 完整值）+ 集成参数（Gateway URL + Verification Type）。

读开发者页聚合（凭据 + 集成参数 + 签名密钥态 + 平台公钥）

返回当前商户开发者页聚合：Merchant Code（明文公开标识）、Gateway Credential 默认脱敏 （末 4 位；展示不触发访问审计；未签发 / 已吊销 → status=unavailable）、 Gateway URL + Verification Type=RSA2（只读集成参数）、商户签名公钥态（registered 携公钥 PEM + keyVersion，或 unset）、 平台公钥（current PEM + keyVersion；未发布 → null）。 完整 Gateway Credential 不在本聚合返回——按需走 revealGatewayCredential（受审计）。 Merchant Code / 平台公钥 / 已登记的商户公钥均非秘密、直接明文返回供 Copy。 聚合任一来源加载失败 → 整体 500、绝不返回部分凭据（前端据此走占位 / 错误态）。

授权披露 Gateway Credential 完整值（reveal / copy；受审计）

按需返回当前商户 Gateway Credential 完整明文（仅当前有效值；历史 / 上一版本值不返回）——受审计。 每次披露均记录访问审计（accessType=reveal|copy）——copy 亦经本端点（避免「已显示明文后再在客户端 copy」的审计盲区）。 先记录访问审计、成功才返完整值；审计写入失败 → 500 不披露（fail-closed）。 响应 Cache-Control: no-store；完整值全链路禁记录（不进日志 / trace / 缓存）。 credential 未签发 / 已吊销 → 409 208014（无明文；overview 已标 unavailable，此为竞态兜底）； 凭据服务暂不可用 → 502/504（fail-closed，不返回部分值）。 非幂等（每次披露刻意各自留痕），不挂幂等键。

signing-keys

签名密钥：商户 RSA2 公钥登记 / 替换（触发轮换宽限期）+ 平台公钥（只读）。

登记 / 替换商户签名公钥（RSA2 PEM；替换即触发轮换宽限期）

登记或替换当前商户的 RSA2 签名公钥（PEM）——供开放 API 验证商户入站请求签名。登记 = 替换 （覆盖当前唯一一把公钥）。并发 / 幂等（判定顺序）：① 提交的公钥与当前公钥相同 → 幂等 no-op 成功（不新 keyVersion / 不重启轮换宽限期 / 不重复审计，与 expectedKeyVersion 无关）；② 与当前公钥不同 → 按 expectedKeyVersion 版本校验：与当前版本一致才替换（生成新公钥），不一致（旧页 / 他人已改）→ 409 208004、前端重载重试。相同公钥重复提交在效果上幂等；不同公钥替换对提交顺序敏感。 服务端先校验可解析为 RSA2 公钥（空 / 非 PEM / 非 RSA2 → 400 204011， 直接拒绝、不产生任何变更）；合法 → 替换生效并 触发轮换宽限期（新旧公钥并验、宽限期后仅新钥），并留密钥变更审计。 活跃轮换宽限期内不接受再次替换（上次替换的旧钥仍在宽限期内 → 409 208015、须待宽限结束；当前公钥与宽限期不受影响；应急强制轮换由平台侧处理、不在本端点）。 判定顺序：与当前公钥相同 → no-op；不同 → 先版本校验（不一致 → 208004）→ 通过后再判活跃宽限期（活跃 → 208015，否则替换）。 本端点只管商户自有公钥（非秘密、可回显）；不提供清除 / 解绑。 不挂幂等键、客户端不应盲目自动重试：对较旧的不同公钥盲目重试会覆盖较新公钥，由 expectedKeyVersion 版本前置校验拦截（不一致 → 409 208004），失败应由用户重新发起。
