Download OpenAPI specification:

收单配置前端契约：Store QR 展示 / 下载、收款页设置（PaymentPageSettings + TipConfiguration）、 账单码（Invoice，列表 / 详情 / 创建 / 作废 / 已支付隐藏）、支付链接（PaymentLink 全部 per-link 配置〔销售内容/生命周期/Policies/Buyer Info〕、创建 / 编辑 / 启停 / 删除 / 卡密池 / Redemptions 只读）、 设备（列表 / 详情 + 绑定/解绑/启停/Regenerate）、设备日志、商户级订单配置（Order Expiry / Customer Service Fee）、POS 小票内容。 字段类型以本 spec 为准。 merchant scope 从会话取、不信请求体；merchant-current-scoped 读写挂 MerchantContextGuard（stale-tab 守卫）。

store-qr

店铺码展示变体 / 下载。

读店铺码展示变体（主 QR + 可用变体）

返回当前商户店铺码可展示变体集（默认主结算币种 Standard 码 + 其余按已开通币种 / QR type / 通道条件显隐； UnionPay QRC 变体需 UPI 通道启用）。 未开通 In-Store Payment Service → 空集。

Authorizations:
cookieAuth
header Parameters
X-Merchant-Context
required
string non-empty

stale-tab 一致性守卫（读写统一）：页面预期的当前商户 ID。与会话当前商户比对，一致才执行； 不一致 → 409 MerchantBffConflict（code 208001、MERCHANT_CONTEXT_CHANGED）——读不返回错商户数据、写不执行，响应 context.currentMerchantId 回显服务端当前商户供前端直接 re-sync。 挂在所有依赖当前商户的读 + 写接口；前端统一 interceptor 发送 + 处理 409（提示并刷新当前商户）。 豁免：listMerchants（返回 currentMerchantId 的 re-sync 入口）、switchCurrentMerchant（切换当前商户）、withdrawOnboardingApplication（目标由 path 指定、与当前商户无关）、以及不依赖当前商户的接口。
