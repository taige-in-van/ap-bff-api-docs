商户后台 React SPA 消费的商户主体资料 / 结算 / Refund PIN / 多商户 / Service Page / 集成 契约。 本服务 OWNS Merchant 聚合根 + BusinessProfile + RefundPin 凭据 + settlement 事前审批 + Service Page 2 层模型。 字段类型以本 spec 为准，业务规则见 requirements。 幂等/CSRF 口径（Q5）:① 创建类 + 非天然幂等写（Add Merchant / Connect AcceptanceMode / Connect Ecommerce / 创建 settlement 变更请求 / 改 Refund PIN / Forgot PIN 重置 / 改 URL 触发重审）挂 Idempotency-Key——无自然业务单号，或重放不自洽（currentPin 已消费、撞 pending → 409），retry 用同键返首次结果; 真天然幂等的纯 PUT/DELETE（改 Business Information / 切当前商户 / 启停 PaymentMethod / Terminal 连接·停用 / Logo）可不挂幂等键; OTP 多步挑战按流程语义每次重新校验。 ② cookieAuth 的非安全方法（POST/PUT/PATCH/DELETE）一律挂 X-CSRF-Token（double-submit，fail-closed）。 ③ merchant scope + stale-tab 守卫（读写统一）：会话 current merchant = scope 唯一权威、不信请求体（switchCurrentMerchant 切换更新会话）；跨 scope 404 反枚举。所有 merchant-current-scoped 读 + 写携带页面预期 X-Merchant-Context（formal = _shared MerchantContextGuard header param；expected merchant ID，仅与会话 current 比对、非 scope selector）——一致才执行；≠ 会话 current → 409 MerchantBffConflict（语义名 MERCHANT_CONTEXT_CHANGED、整数 code；读不返回错商户、写不执行；context.currentMerchantId 回显〔非 details〕）。前端统一 interceptor 发送 + 处理 409（提示 + 刷新当前商户；BroadcastChannel/focus 只提前发现）。治理仅 cookieAuth BFF 面 + 依赖当前商户 op；不适用 .internal / 公开 / 设备面 / 不依赖当前商户接口。豁免 listMerchants / switchCurrentMerchant / withdrawOnboardingApplication（path-selector、target≠current）/ 全局接口。 ④ Idempotency-Key 语义：retry 用同一 key 返回首次结果（retry-safe），区别于 OTP token 的一次性消费 anti-replay（消费后不可重取首次结果）。 ⑤ OTP 错误分类：错误但可重试 → 400（ErrorEnvelope.details.attemptsRemaining）;尝试超限锁定 → 423;过期 / 已消费 → 410;Resend 冷却 → 429。

profile

BusinessProfile 概览与编辑：Profile 聚合（Header card + 4 section + 资金 read model + Locked 标识）、Business Information 7 可编辑字段直接生效、Logo 上传/移除。Locked 字段服务端 enforce（绕过 → 403）。(FR-01/02/03)

查看 Profile 概览（Header card + 4 section；聚合 BusinessProfile + Merchant + AUTH Contact + FINACCT 资金）

聚合当前选中 ACTIVE 商户的 Profile 概览：Header card（Logo / Trade Name (DBA) / Legal Business Name / Industry • MCC / Merchant ID） + 4 section（Business Information / Settlement / Contact Information / Documents 摘要）。Locked 字段附 locked: true 标识（FR-03）; account_number 掩码仅末 4 位（•••• 5678）;资金字段（Account Balance / Refund Reserved）来自 FINACCT read model（只读消费）; Contact Information 同源 AUTH（无本域 copy）。当前商户非 ACTIVE → 不返回 Profile（前端按 Merchant 状态分流到入驻向导 / 审核状态页）。
