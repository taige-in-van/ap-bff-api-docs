商户后台 交易管理 (20-05 BFF) — 前端契约面 (0.4.1)

Download OpenAPI specification:

商户后台交易域的读 + 命令窗口：统一支付交易查询表（多维过滤 + 汇总卡）· 交易详情（含回调技术细节 + 换汇呈现）· 统一退款查询表 · 退款详情 · 支付 / 退款收据（PDF）· 数据导出（Excel）· 发起退款（凭 Refund PIN）· 撤销待处理退款 · 预授权完成（Release）。 交易 / 退款 / 预授权的状态机、清结算、Fee、通道交互归支付核心系统，本域只提供「可见 + 可操作」窗口。 金额一律最小币种单位整数 + currency；列表行不同标价币种不混算，汇总卡单一结算币种（不按标价币种分组）；换汇信息在后端返回换算信息（通道返回汇率 / 换算信息）时有条件展示——标价 ≠ 结算币种为典型场景、非判定条件。 全部端点前置 Merchant.status = ACTIVE 门禁（非 ACTIVE → 403；仅 ACTIVE 商户可用日常能力，会话上下文含 INCOMPLETE / PENDING_REVIEW / REJECTED 仅供入驻 / 拒绝页）。字段类型以本 spec 为准。

payments

支付交易：查询表 / 汇总卡 / 详情 / 收据 / 导出（读）+ 发起退款 / 预授权完成（命令）。

统一支付交易查询表（过滤 + 排序 + 分页）

返回当前商户支付交易列表（默认「今日」时间窗）。多维过滤（时间 / 支付方式〔品牌 Family〕/ 来源 / 状态）+ 关键词（订单号 / 备注模糊，金额数值精确、跨标价币种按数值命中）+ 排序 + 分页。 每行金额按其标价币种展示、不同币种不合并；行操作可见性（可退 / 收据）由后端返回。查询超时 → 降级（展示「暂不可用」，不返回半截数据）。

Authorizations:
cookieAuth
query Parameters
from
string <date>

起始日（商户 IANA 时区解析为本地日、服务端转 UTC 半开区间 [start, end) 查询——DST / 跨午夜安全；省略 = 今日）。列表 / 汇总 / 导出共享同一查询契约。

to
string <date>

截止日（商户 IANA 时区、含当日；切日语义同 from）。

paymentMethod
string

支付方式筛选（稳定 Family code，如 wechat / alipay——非本地化展示名；候选集 = filterOptions.paymentMethods[].code）。

source
string

来源筛选（稳定来源 code，如 store_qr——非展示文案；后端动态配置枚举、候选集 = filterOptions.sources[].code）。退款侧 = 原支付来源（≠ 退款来源 refundSource——后者服务端用于排除 system_auto、不作 FE 筛选值）。

status
string (PaymentStatus)
Enum: "paid" "partially_refunded" "refunded" "failed" "PREAUTH_HOLDING"

支付订单商户可见状态（小写 = 商户口径 / 大写 = 内核态透传）：paid 已收款 · partially_refunded 部分退款 · refunded 全额退款 · failed 失败 · PREAUTH_HOLDING 预授权挂起。 paid / partially_refunded / refunded 均基于内核 PAID（后两者为退款度派生 f(成交, Σ已成功退款)、非独立 status）；failed 来自内核 FAILED；PREAUTH_HOLDING 为内核挂起态透传。瞬态 CREATED / PAYING 不进列表。 封闭 enum：新增取值 = 破坏性变更、走契约版本化；消费方须 unknown-safe 处理未识别值（保守对待、不静默丢弃）。

keyword
string

关键词：订单号 / 备注模糊；纯数值按金额精确匹配（跨标价币种按数值命中、不区分币种）。

sort
string (PaymentSort)
Enum: "time_desc" "time_asc" "amount_desc" "amount_asc"

支付列表排序（默认时间倒序）。封闭 enum：新增取值走版本化；消费方 unknown-safe。

pageNum
integer >= 1
Default: 1

页码(从 1 计;分页)

pageSize
integer [ 1 .. 200 ]
Default: 20

每页记录数(上限可配置;分页默认 20)

header Parameters
X-Merchant-Context
required
string non-empty

stale-tab 一致性守卫（读写统一）：页面预期的当前商户 ID。与会话当前商户比对，一致才执行； 不一致 → 409 MerchantBffConflict（code 208001、MERCHANT_CONTEXT_CHANGED）——读不返回错商户数据、写不执行，响应 context.currentMerchantId 回显服务端当前商户供前端直接 re-sync。 挂在所有依赖当前商户的读 + 写接口；前端统一 interceptor 发送 + 处理 409（提示并刷新当前商户）。 豁免：listMerchants（返回 currentMerchantId 的 re-sync 入口）、switchCurrentMerchant（切换当前商户）、withdrawOnboardingApplication（目标由 path 指定、与当前商户无关）、以及不依赖当前商户的接口。

Responses

商户后台 BFF（兼前端 BFF）面；浏览器 cookieAuth，非跨部署服务消费面 / 非外部面。

交易页头汇总卡（随筛选、单一结算币种）

返回与 listPayments 同筛选上下文的汇总卡：交易笔数（币种无关计数）+ 单一结算币种（标价按成交汇率折算入商户结算币种、不按标价币种分组）的 Gross Sales / Refund Amount / Net Sales + To Settle。 To Settle 为账户账务的资金派生值、不由交易筛选条件重算（结算币种口径）。

导出支付交易（按当前筛选全集 Excel）

按当前筛选 / 搜索 / 排序条件命中的全集（非当前分页）导出当前商户支付交易 Excel。列头 / 枚举 / 金额 / 日期按 locale 本地化（locale 来源 = Accept-Language（BCP-47）→ 回退会话 locale → 商户默认；响应回 Content-Language）、金额带币种。 导出结果集上限为业务参数（部署可配置）；超过上限 → 400·204009 拒绝 + 提示缩小筛选范围（不截断、不异步、非上传体过大 413），商户收窄条件后重导。

发起退款（全 / 部分 + Refund PIN）

对可退订单发起全额 / 部分退款——可退资格 = 后端权威结果（paid / partially_refunded ∧ 剩余可发起额 > 0，或平台挂起 PREAUTH_HOLDING 经预授权放行）；BFF 不硬编码允许状态，最终以后端重验为准。金额以成交结果金额币种为准、须 ∈ (0, 剩余可发起额]（= 成交 − 已成功 − 在途占用）。 凭商户级 Refund PIN 授权，PIN 校验结果映射为响应：通过 → 继续；不匹配 / 不可校验 → 403·206003（反枚举同响应、不披露是否匹配）；连续失败锁定 → 423·206004（防爆破）；数据完整性故障 → 5xx。原始 PIN 仅入参、绝不回显。建单后由平台审核规则判定 REVIEWING / CHANNEL_PROCESSING（SUBMITTED 仅建单阶段瞬态、不回显）。 允许与未终态退款并发（各占用其额）。退款原因系统默认 customer_request、无商户 picker。 幂等：同 Idempotency-Key + 同 payload → 返冻结的首次结果、不重复创建退款单；同键异 payload → 409（208005）；处理中 → 409·208011（FE 以同键同 payload 重发 createRefund 轮询、无独立查询端点）。退款单已创建即回 refundNo + refundStatus（首次受理为非终态；一经成功即冻结、后续同键重发返冻结结果），终态经后续刷新收敛（FE 勿误读为最终成功）。可退额校验权威在后端：BFF 侧剩余可发起额仅 UX 预检（可能陈旧）；后端在受理时原子重验资格 / 币种 / 剩余额并占用——并发两请求之和超限时仅可容纳部分成功、其余以 400 + reason（机器判别）+ detail（人类文案） 拒绝（可退不足 / 能力关闭 / 产品已核销 / 挂起受理面未开启 均 400、**非 409**；FE 面 {code 204008, reason, detail}——204007 仅 BFF amount 格式预检〔无 reason〕；FE 据 reason enum 区分、重取剩余额后可再退），杜绝并发超退资损。
