商户后台 账户账务与清结算 (20-06) — 前端契约面 (0.5.8)

Download OpenAPI specification:

商户后台 React SPA 消费的资金概览 / 账户流水 / 结算单 / 月度账单 / 退款预留金管理 / 数据导出 契约。 资金概览 / 流水 / 结算单 / 月度账单为只读；结算金额由平台资金能力产出，本面只展示、不二次计算。 储备金 Top Up / Withdraw 为资金动作，结算影响归清结算能力；auto-replenish 配置由本面管理、其自动补足由平台触发；Owner email OTP 二次验证经身份验证能力。 金额一律商户唯一结算币种（单商户单结算币种）；金额最小币种单位整数字符串（components Money）。字段类型以本 spec 为准。 幂等 / CSRF 口径:① 有副作用的写（储备金 funding / withdrawal、导出创建）挂 Idempotency-Key（资金路径强一致，重投返原结果）;OTP 挑战类按流程语义不挂幂等键。 ② cookieAuth 的非安全方法（POST/PUT/DELETE）一律挂 X-CSRF-Token（double-submit，fail-closed）;无公开端点（资金面全程受 RBAC + 会话）。 鉴权 / 行级授权:所有读写均限会话当前 merchant scope（merchant 上下文从会话取、不信请求体）;储备金 funding（Top Up / auto-replenish）经 Owner email OTP（身份验证能力，资金动作二次验证）放行,Withdraw 不要求 OTP（仅内部回 Account Balance、不外流）。 状态码约定（同商户后台其它 BFF 面）:会话失效 / 未认证由边缘网关 + 会话层统一 401,业务面端点只表达 403（无权限 / scope 不符 / OTP 失败 / fail-secure），故各端点列 403 不列 401（有意一致省略）;跨 scope 资源统一 404 防枚举。

account-overview

资金概览：Account Balance / To Settle（派生展示）/ Refund Reserve / 可用额，商户结算币种。只读。

资金概览（Account Balance / To Settle / Refund Reserve / 可用额）

资金概览（商户 scope）：Account Balance（账本值恒 ≥ 0）、To Settle（按手续费结算模式派生、本面不二次计算）、Refund Reserve、可用额（= Account Balance − 应收商户款,可呈负,纯展示）。 Account Balance 卡片展示值是否并入平台侧预授权挂起额由管理后台设置的 per-商户展示配置控制（默认并入,本面只展示、不改账本值）。上游账务查询超时 → degraded=true 降级展示"暂不可用"、不伪造资金,不阻塞页面。

Authorizations:
cookieAuth
header Parameters
X-Merchant-Context
required
string non-empty

stale-tab 一致性守卫（读写统一）：页面预期的当前商户 ID。与会话当前商户比对，一致才执行； 不一致 → 409 MerchantBffConflict（code 208001、MERCHANT_CONTEXT_CHANGED）——读不返回错商户数据、写不执行，响应 context.currentMerchantId 回显服务端当前商户供前端直接 re-sync。 挂在所有依赖当前商户的读 + 写接口；前端统一 interceptor 发送 + 处理 409（提示并刷新当前商户）。 豁免：listMerchants（返回 currentMerchantId 的 re-sync 入口）、switchCurrentMerchant（切换当前商户）、withdrawOnboardingApplication（目标由 path 指定、与当前商户无关）、以及不依赖当前商户的接口。

账户流水查询（过滤 + 搜索 + 分页，只读）

账户流水（商户 scope）：按时间倒序分页;每行含资金池 / 类型 / 带符号金额 / 对应资金池变动后余额 / 关联单据或操作记录。空结果返回空态、不报错。默认时间窗避免空窗（业务参数可配）。

settlement

结算单（Settlement）：按清算周期 / 状态查列表 + 逐笔结算影响明细详情。只读。

结算单列表（周期 / 状态过滤 + 搜索 + 排序 + 分页）

结算单列表（商户 scope）。日期倒序默认;状态过滤为商户视角四态（待结算 / 处理中 / 已结算 / 失败,不展示具名冻结态）。 FAILED 行标记并提示资金退回待结算池、后续重新归集或人工处置。默认窗覆盖近期有数据范围（业务参数可配）。

退款预留金管理：Top Up（手动充值）/ auto-replenish 配置（均经 Owner email OTP）/ Withdraw（取回 Account Balance，无 OTP）。OTP 经身份验证能力，结算影响归清结算能力。

发起 Owner email OTP 挑战（绑定本次操作摘要）

向该商户 Owner 的已验证邮箱发起一次性 email OTP（经身份验证能力）。挑战绑定本次操作摘要（Top Up 金额 + auto-replenish 配置的开关 / 阈值 + 字段存在性）与发起会话——提交 funding 时须摘要一致、同会话提交,否则 → 403（防串改金额 / 币种 / 夹带配置）。操作上下文（商户 / Owner）取会话 scope、不信请求体（无有效 scope → 403 不发起）。返回 challengeId + 脱敏投递目标 + 冷却。展示摘要与绑定摘要同源（防 Owner 看小额、放大额）。OTP 有效期 / 冷却 / 错误尝试上限为业务参数（默认可配）。挑战类不挂幂等键（每次重新发起）。Top Up 与 auto-replenish 同次共用一次 OTP（摘要含配置全字段,Owner 知悉同时变更自动补足规则）。
