商户后台 账户账务与清结算 (20-06 / D-MERCH BFF) — 前端契约面 (0.5.3)

Download OpenAPI specification:

商户后台 React SPA 消费的资金概览 / 账户流水 / 结算单 / 月度账单 / 退款预留金管理 / 数据导出 契约。 本服务是投影读 + 薄写编排层：读经 JOIN 上游发布视图（10-05 余额 / 流水、10-06 结算单、10-07 月度账单）， 写经编排:储备金资金动作 → 10-05 reserve-movements（Top Up / Withdraw / auto-replenish 执行）、auto-replenish 配置 → FINACCT 本地 refund_reserve_config、Owner email OTP → 40-06 step-up； 本服务不自落资金分录、不自管 OTP、不重定义资金口径，OWNS 仅 D-MERCH 本地：展示配置 / 导出任务 / 储备金操作发起记录（BFF 侧，非权威资金）。 金额一律商户唯一结算币种（单商户单结算币种，骨干 11）；金额最小币种单位整数字符串（components Money）。字段类型以本 spec 为准，业务规则见 requirements。 幂等 / CSRF 口径:① 有副作用的写（储备金 funding / withdrawal、导出创建）挂 Idempotency-Key（资金路径强一致，重投返原结果）;OTP 挑战类按流程语义不挂幂等键。 ② cookieAuth 的非安全方法（POST/PUT/DELETE）一律挂 X-CSRF-Token（double-submit，fail-closed）;无公开端点（资金面全程受 RBAC + 会话）。 鉴权 / 行级授权:所有读写均限会话当前 merchant scope（merchant 上下文从会话取、不信请求体）;储备金 funding（Top Up / auto-replenish）经 Owner email OTP（40-06 资金动作 step-up /step-up/fund-actions,BFF 编排）放行,Withdraw 不要求 OTP（仅内部回 Account Balance、不外流）。 状态码约定（同 20-01 BFF）:会话失效 / 未认证由边缘网关 + 会话层统一 401,业务面端点只表达 403（无权限 / scope 不符 / OTP 失败 / fail-secure），故各端点列 403 不列 401（有意一致省略）;跨 scope 资源统一 404 防枚举。

account-overview

资金概览：Account Balance / To Settle（派生展示）/ Refund Reserve / 可用额，商户结算币种。只读投影 10-05 v_account_balance_v1。(FINACCT-FR-01)

资金概览（Account Balance / To Settle / Refund Reserve / 可用额）

投影 10-05 v_account_balance_v1（商户 scope）：Account Balance（canonical 账本值恒 ≥ 0）、To Settle（D-CORE 按手续费结算模式派生、本域不二次计算）、Refund Reserve、可用额（= AB − 应收商户款,可呈负,纯展示）。 Account Balance 卡片展示值是否并入平台侧预授权挂起额由管理后台设置的 per-商户展示配置控制（默认并入,20-06 只读消费、不改 canonical 账本值）。上游超时 → degraded=true 降级展示"暂不可用",不阻塞页面（FR-01-AC-04）。

Authorizations:
cookieAuth
header Parameters
X-Merchant-Context
required
string non-empty

merchant-current-scoped 请求守卫（stale-tab 一致性，读写统一）：页面预期商户 ID（expected merchant）。仅与会话 current merchant 比对、非 scope selector—— 会话 current 仍是 merchant scope 唯一权威；一致才执行；≠ 会话 current → 409 MerchantBffConflict（语义名 MERCHANT_CONTEXT_CHANGED，整数 code 208001、SSOT 见 20-商户后台/api/error-codes.yaml）——读不返回错商户数据、写不执行； context.currentMerchantId（非 details 字段校验数组）回显服务端会话当前商户供前端直接 re-sync。统一挂在各 module 所有 merchant-current-scoped 读 + 写 operation； 前端统一 interceptor 发送 + 处理 409（提示并刷新当前商户；BroadcastChannel / focus 只作提前发现、改善体验）。 治理范围仅 cookieAuth 浏览器 BFF 面 + 依赖当前商户的 op；不适用 .internal serviceAuth 面 / 公开 register·login·callback（security:[]）/ 设备面 / 其他不依赖当前商户的接口。BFF 面内豁免：listMerchants（re-sync 发现入口、返回 currentMerchantId）、switchCurrentMerchant（设当前 authority changer）、withdrawOnboardingApplication（path-selector：target 由 path 定、与会话 current 无关）、全局接口。

账户流水查询（过滤 + 搜索 + 分页，只读分录投影）

settlement

结算单（Settlement）：按清算周期 / 状态查列表 + 逐笔结算影响明细详情。只读投影 10-06 v_settlement_summary_v1 / v_settlement_line_v1。(FINACCT-FR-03/04)

结算单列表（周期 / 状态过滤 + 搜索 + 排序 + 分页）

投影 10-06 v_settlement_summary_v1（商户 scope，L2）。日期倒序默认;状态过滤为商户视角四态投影（待结算 / 处理中 / 已结算 / 失败,UNCONFIRMED 并入处理中,不展示具名冻结态）。 FAILED 行标记并提示资金退回待结算池、后续重新归集或人工处置。默认窗覆盖近期有数据范围（业务参数可配 X-07）。

按当前查询条件（筛选 / 搜索 / 排序）创建导出任务,范围与查询一致、不局限当前分页（FR-09-AC-01/03）。导出 scope = 发起会话 merchant scope(持久化进 export_job.scope_snapshot 作 scope 上界冻结、只防放宽;worker 执行复用同一行级授权 chokepoint,不在执行时重信 filters / resourceId 放宽 scope;resourceId 跨 scope → 拒绝/404)。导出是绕分页的批量读路径——实时授权分阶段:worker 异步执行 = 发起 operator 的 membership / 读权限当前有效(40-06 在线求值、与活跃会话解耦;operator 登出 / 会话自然过期不拒执行,仅 membership 撤销 / 降权拒);同步下载 = 会话 + membership 双裁决。即「scope_snapshot 不放宽 ∩ 实时 membership 不越当前权限」取交集——撤权 operator 不能带走数据,又不误伤登出后的正常异步任务。详 design §安全落地。大结果集异步处理、超业务参数上限不静默丢弃而给超限提示或转异步（FR-09-AC-07）;导出请求记审计事件（X-03,FR-09-AC-08）;空结果导出空表头文件或提示无数据。OWNS 导出任务（D-MERCH 本地 export_job）。幂等：挂 Idempotency-Key、同键重投返首次创建的 export_job（不重复建任务、响应丢失可安全重试）；同键异 scope/参数 → 409。
