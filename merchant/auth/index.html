商户后台 注册登录与认证 (20-01 / D-MERCH BFF) — 前端契约面 (0.4.2)

Download OpenAPI specification:

商户后台 React SPA 消费的注册 / 登录 / 会话 / 账户安全 / 团队成员 契约。 本服务是 OAuth2 client：认证 / 凭据 / RBAC / 撤销 经 REST 调 40-06 身份服务， 本服务 OWNS Operator 商户成员 profile + BFF 会话 + 注册校验物。字段类型以本 spec 为准，业务规则见 requirements。 幂等/CSRF 口径:① 创建类(注册/建成员)与非天然幂等 / 跨单写者 / 有副作用的写(如改密〔失效会话〕、改成员〔本地 Operator + 转发 40-06 跨写〕)挂 Idempotency-Key;真正天然幂等的纯 PUT/DELETE 可不挂;认证类(登录/登出/MFA/扫码)与多步 OTP 挑战类(改邮箱/找回)按流程语义不挂幂等键(每次都应重新认证/校验)。 ② cookieAuth 的非安全方法(POST/PUT/DELETE)一律挂 X-CSRF-Token(double-submit,fail-closed);公开端点(security:[])无会话故无 CSRF。

FR-01 邮箱激活（确认页主动点击触发的 POST；防扫描器 prefetch）

校验一次性验证 token（仅在 membership 成功提交时消费;之前各步失败 token 不消费、可用同 token 重试,避免卡在 PENDING 但 token 已失效）→ 编排激活,membership 最后提交:先本地事务备 ONB Application(DRAFT) + Merchant Shell + Operator 就绪（此刻仍无 membership → 登录仍 gate）,成功后才调 40-06 provisionMember 建 active membership（= 启用登录的提交点 + 消费 token）。 membership 之前任一步失败 → 不返回成功、Operator 仍 PENDING、可重试（AC-11）;membership 成功后才返回 ACTIVE。验证链接的 GET 确认页是前端路由、不激活账号，故不在本 API。

session

登录与会话生命周期：密码登录（MFA 联动）、WeChat 扫码登录、会话校验 / 身份摘要 / 可见菜单、登出。浏览器持 httpOnly 会话 cookie，非安全方法带 X-CSRF-Token。(FR-02/03/05)

FR-02 密码登录（邮箱或用户名 + 密码；MFA 联动；签发 BFF 会话）

BFF 调 40-06 认证。未启 MFA → 建 BFF 会话（set-cookie）。已启 MFA → 返回 MFA_REQUIRED + 短时 mfaChallenge（不建会话）。 PENDING_VERIFICATION（自助注册待验）/ INVITED（邮箱员工待设密,前端提示「请用邀请邮件设密激活」）→ 无 active membership 不签发会话;锁定 423;商户 suspended / 账号禁用 403。防爆破计数归 40-06（FR-04 / INV-14）。
